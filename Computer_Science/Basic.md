# 计算机工程世界观

## 1. 什么是计算机

站在工程的角度，计算机是一种：能够在物理世界中，以确定性的规则，对信息状态进行表示、存储、转换和控制的工程系统。

让我们对这句话进行拆解：

### 工程系统

一个工程系统，必须满足：
- 可预测
- 可重复
- 可控制
- 可扩展

计算机最重要的不是“智能”，而是**极度可控**

### 信息状态

- 状态 --> 系统在某一时刻内的全部内部情况
- 信息 --> 对状态的抽象描述

计算机只进行**状态的操纵**。

### 确定性规则

这是**工程的底线**：

- “同样的初始状态，必须得到同样的结果。”

没有确定性，就无法调试、无法复现、无法扩展。


## 2. 信息、状态与二进制

计算机世界的一切，本质上都是**状态 + 规则 + 约定**。在计算机中，没有“天然正确”，只有“设计选择”。

### 信息

在工程领域里，信息是**可被区分的状态**，只要两个状态可以被可靠的区分，它们就可以承载信息。

### 状态

计算机中一切的高级概念都建立在状态之上。

工程系统的核心目标是：**控制状态的变化**。因此，一个“合格的工程状态”必须：

- 可区分
- 可保持
- 可转换
- 可复现

### 二进制

在现实中，世界充满噪声，精确测量成本极高，但在工程中**“稳定性”远远比“表达力”更重要**。

于是工程师做了一个关键的决策：**牺牲表达效率，换取稳定性。**

#### 二进制的工程优势

| 维度 | 二进制 |
|---|---|
| 抗干扰 | 极强 |
| 判定成本 | 极低 |
| 实现难度 | 极低 | 
| 可扩展性 | 极高 |

所以，0 和 1 并不是“少”，而是“稳”。

### 比特 (bit) 

1 bit = **一个可以被可靠区分的二态系统**

因此，bit 是**工程单位**， 而不是数学概念或者数字

### 状态组合：复杂性的来源

单个 bit 很简单，但工程的力量来自于**大量简单状态的组合**。

>1 bit 只有两种状态  
>8 bit 有256种状态  
>32 bit 大约有43亿种状态  
>64 bit 在工程上“几乎无限”  

### 信息 ≠ 含义

计算机内部只有：

- 0
- 1
- 状态变化

而含义存在于人类的头脑中。因此：**含义来自约定，而不是机器理解**

### 编码

编码是人类赋予状态组合以意义的规则，是一种工程协议。
